# Commands Directory Contract

## Why this exists
- 명령 실행을 deterministic 계약으로 고정한다.

## What belongs here
- 각 명령의 의도/입력/실행 프로토콜/산출물/검증/실패 처리/종료 조건.

## What does NOT belong here
- 특정 제품 도메인 로직 상세.

## Typical mistakes
- 명령 문서가 체크리스트 없이 서술형으로만 작성됨
- 실패 시 재시도 규칙 부재

## Preventive guidance
- 모든 명령 문서에 동일한 8개 섹션을 유지한다.
